- **事件驱动架构**：后端以 `EventBus` 为核心神经中枢，解耦智能体（Agents）、研究引擎（Engines）与工具集（Tools），所有状态变更均通过事件发布与订阅触发。
- **ATA编排调度**：`ATAOrchestrator` 作为全局单例，监听事件总线并管理每个“大脑”的独立思考循环（Brain Loop），根据收敛压力动态调度不同角色的智能体进行探索或综合。
- **前后端数据契约**：前端通过统一的 API 层（`api.ts`）与后端交互，后端将复杂的认知元素（Cognitive Elements）与关系聚合为力导向图所需的 Nodes/Edges 结构，支持实时知识图谱渲染。
- **共享持久化层**：所有模块（后端逻辑、事件审计、前端查询）共用基于 SQLite 的 `database.py` 模块，确保认知状态、事件日志与用户数据的一致性。
