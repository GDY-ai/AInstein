- **事件驱动架构**：以 EventBus 为核心通信枢纽，连接 agents（智能体）、engines（研究引擎）与 orchestrator（ATA 编排器），实现组件间的解耦与异步响应。
- **双轨数据持久化**：database.py 统一管理旧版项目表与新版硅基大脑 Schema（认知元素/关系/博弈），engines/three_round.py 实现了从传统研究结果到认知图谱的双写逻辑。
- **统一入口与生命周期管理**：app.py 提供 RESTful API 暴露大脑创建、博弈触发及状态查询；wsgi.py 负责应用启动时的数据库初始化、定时任务锁竞争及活跃大脑的思考循环恢复。
- **动态角色注册机制**：agents/framework.py 中的 RoleRegistry 将静态角色定义持久化至数据库，支持智能体在运行时根据性格向量与视角偏好动态生成 Prompt。
